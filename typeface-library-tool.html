Typeface Library Tool

Roles

  • Designer

Tools

  • Figma
  • FigJam

I designed a typeface library tool to make it easier for junior designers to find, test, and compare typefaces.

I know from personal experience that finding the perfect typeface for a project can be difficult and time-consuming. In addition to combing through pages of different typefaces, designers often have to constantly switch between the typeface library and their design software to test how different typefaces look in the context of their project. Other designers I spoke with shared this sentiment, so I decided to redesign and improve the experience of searching for typefaces.

Research

How do other designers find typefaces?

While I had specific improvements that I would personally want in the typeface-finding process, I wanted to know the workflows and problems that other designers have, since I am not designing a tool for just myself.

Potential user groups

After talking to other designers and doing secondary research, I settled on three potential user groups that go through the process of finding typefaces.

Validating my user flow

Based on my research, I mapped out the user journey of a new designer searching for a typeface. To ensure my understanding of the process was correct, I showed it to some design students to see if it matched their experiences.

Key features and design iterations

A search page built for easy scanning

Features

After looking at the existing typeface library websites, there were some features that I thought might improve usability based on my research and user interviews, such as:

  • More granular and specific filters that give correct and consistent results as well as fewer subjective filters, such as a filter for "fun" typefaces. This would give a designer more fine-tuned control over their results.
  • Options to filter for typefaces suited to particular mediums and types of design work.
  • A way to control the settings for how the sample text is displayed (e.g. the line height, the weight, etc.) to allow a designer to quickly see if a certain typeface works in the context that they need.

User testing

When I tested this interface with a user, she said that it was a little cramped, and would rather have more room for the typefaces themselves. Based on that feedback, I removed the right sidebar and added a switch to the left sidebar that would display the content formerly on the right.

Turning a filter into a feature

Before user testing

Since one of the biggest factors in the typeface a designer chooses is the type of design work they are doing, I added that to the filters to help designers narrow down their search.

User feedback

The designer I tested with said that, while she liked the idea of being able to easily find typefaces that are good for specific types of design work, she wondered what would happen if she selected a filter in the "design work" section that logically wouldn't make sense alongside a selected filter in another section.

My solution

Instead of having the type of design work as a single filter, I implemented presets for specific combinations of filters. This would make it easier for a designer to quickly find typefaces suited for certain types of design work while ensuring that the user doesn't get zero search results since the filters are predefined.

Lessons learned

Think about the technology

While I didn't develop this into a live app, I did think about some of the technical constraints that I might face if I were to do that, such as how to load all the typefaces without slowing down the application.

Information overload

After doing user testing, one of the main pieces of feedback was that there was too much information displayed on each screen. That showed me that not everything always needs to be displayed in the same place.
